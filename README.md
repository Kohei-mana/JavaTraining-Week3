# JavaTraining-Week3
タスク管理アプリを作成しよう

目的：タスクの登録、一覧表示、削除、完了を行う。前回学習したものと、例外処理、コレクション、ラムダ式を新たに学習。

1. 実行環境と手順
javac src/*.java　→　java -classpath src TaskApp

2. クラス設計
・Task.java（データクラス）
タスクのタイトル、期限、完了状態をカプセル化して保持する。

・TaskManager.java（処理クラス）
タスクのリスト（ArrayList）を保持し、登録、一覧表示、削除、完了操作の処理を行う。
forEach、ラムダ式を使用

・TaskApp.java（メインクラス）
メニュー表示、ユーザー入力受け付け、try-catchによる入力例外処理を行い、TaskManagerへ処理を行わせる。

3. 工夫点と学んだこと
・ラムダ式とコレクションの活用
ArrayListに対して、forEachメソッドとラムダ式を使用し、タスクの一覧表示を簡潔に記述した。

・try-catchの実装
TaskApp.javaのメインループ内で、メニュー選択のscanner.nextInt()をtry-catchで囲むことで、ユーザーが数字以外を入力してもプログラムが終了せず、再入力を促すことができる。

・ユーザー視点での使いやすさ
前回指摘された、削除操作を行うとき、（今回）完了操作を行うときに、タスクを一覧表示することで、ユーザーがどのIDを入力すればいいかがわかりやすくなる。
